## 一、功能测试

### 1.1 新增打卡功能

#### TC-ADD-001 正常新增打卡
- **前置条件**：用户已登录，进入打卡页面
- **测试步骤**：
  1. 点击首页右上角"+"按钮进入打卡页面
  2. 输入学习内容："学习了微信小程序云开发"
  3. 输入学习时长：60（分钟）
  4. 确认日期为当前日期
  5. 点击"提交打卡"按钮
- **预期结果**：
  - 提示"打卡成功"
  - 自动跳转回首页
  - 首页列表显示新增的打卡记录
  - 统计卡片数据更新（累计打卡天数+1，总时长+60分钟）

#### TC-ADD-002 修改日期后提交打卡
- **前置条件**：用户已登录，进入打卡页面
- **测试步骤**：
  1. 点击日期选择器，选择昨天日期
  2. 输入学习内容："补打卡昨天的学习"
  3. 输入学习时长：30（分钟）
  4. 点击"提交打卡"按钮
- **预期结果**：
  - 打卡成功，记录日期为昨天
  - 首页列表按日期倒序排列，昨天的记录在今天记录之后

#### TC-ADD-003 打卡内容包含特殊字符
- **前置条件**：用户已登录，进入打卡页面
- **测试步骤**：
  1. 输入学习内容："学习#特殊字符@测试&符号*"
  2. 输入学习时长：45（分钟）
  3. 点击"提交打卡"按钮
- **预期结果**：
  - 打卡成功，特殊字符正常保存和显示

#### TC-ADD-004 打卡内容包含表情符号
- **前置条件**：用户已登录，进入打卡页面
- **测试步骤**：
  1. 输入学习内容："今天学习很开心😊🎉"
  2. 输入学习时长：50（分钟）
  3. 点击"提交打卡"按钮
- **预期结果**：
  - 打卡成功，表情符号正常保存和显示

## 四、数据测试

### 4.1 数据持久化

  - 数据完全一致
  - 统计数据一致

#### TC-DATA-003 多设备数据同步
