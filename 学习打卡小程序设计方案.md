## 一、项目概述

### 1.1 产品定位
一款面向校园学生的学习打卡小程序，帮助用户记录每日学习情况，养成良好学习习惯。

### 1.2 核心功能
| 功能模块 | 说明 |
|---------|------|
| 打卡记录 | 记录学习内容、时长、日期 |
| 打卡列表 | 查看所有历史打卡记录 |
| 打卡详情 | 查看单条打卡的详细信息 |
| 数据统计 | 展示打卡天数、总时长等统计信息 |

### 1.3 核心机制对照（方向二要求）
- ✅ 明确的输入 → 处理 → 输出流程
- ✅ 数据使用云数据库存储，换设备登录数据不丢失
- ✅ 至少包含2个相关功能页面（打卡列表页 + 打卡详情页）
- ✅ 有基本的数据增删改查操作

## 三、数据库设计

### 3.1 集合：`checkin_records`（打卡记录）

#### 字段定义
| 字段名 | 类型 | 说明 | 是否必填 |
|--------|------|------|---------|
| `_id` | String | 系统自动生成，记录唯一标识 | 是 |
| `_openid` | String | 用户唯一标识（系统自动添加） | 是 |
| `content` | String | 打卡内容（学习了什么） | 是 |
| `duration` | Number | 学习时长（分钟） | 是 |
| `date` | String | 打卡日期，格式：YYYY-MM-DD | 是 |
| `createTime` | Date | 创建时间 | 是 |
| `updateTime` | Date | 更新时间 | 否 |

#### 集合权限设置
```
权限模式：仅创建者可读写
说明：每个用户只能查看和管理自己的打卡记录
```

#### 索引设置
| 索引字段 | 索引类型 | 说明 |
|---------|---------|------|
| `_openid` | 普通索引 | 加速按用户查询 |
| `date` | 普通索引 | 加速按日期查询 |
| `_openid + date` | 复合索引 | 加速按用户和日期组合查询 |

#### 示例数据
```json
{
  "_id": "auto-generated-id",
  "_openid": "user-openid-xxx",
  "content": "学习了微信小程序云开发，完成了数据库设计",
  "duration": 60,
  "date": "2026-06-10",
  "createTime": "2026-06-10T10:00:00.000Z",
  "updateTime": "2026-06-10T10:00:00.000Z"
}
```

## 八、开发步骤引导

### 步骤 1：配置云环境

1. 打开 `miniprogram/app.js`
2. 将 `env` 字段设置为：`cloud1-d6gil20hm00cbde86`

### 步骤 2：创建数据库集合

在微信开发者工具中：
1. 点击顶部工具栏「云开发」按钮
2. 进入「数据库」面板
3. 点击「+」创建集合
4. 集合名称：`checkin_records`
5. 权限设置：选择「仅创建者可读写」
6. 创建索引：
   - 添加 `_openid` 索引（普通索引）
   - 添加 `date` 索引（普通索引）

### 步骤 3：创建云函数

在微信开发者工具中：
1. 右键 `cloudfunctions` 文件夹 → 「新建 Node.js 云函数」
2. 依次创建以下云函数：
   - `checkinAdd`
   - `checkinList`
   - `checkinDetail`
   - `checkinUpdate`
   - `checkinDelete`
3. 每个云函数创建后，右键 → 「上传并部署：云端安装依赖」

### 步骤 4：创建页面

1. 在 `miniprogram/pages/` 下创建：
   - `checkin/` 文件夹（包含4个文件）
   - `detail/` 文件夹（包含4个文件）
2. 更新 `app.json` 中的 `pages` 配置

### 步骤 5：编写页面逻辑

按照本方案第五部分的页面设计，编写各页面的：
- WXML（页面结构）
- WXSS（页面样式）
- JS（页面逻辑，调用云函数）

## 十、注意事项

1. **云环境 ID**：所有云函数和小程序端初始化时必须使用 `cloud1-d6gil20hm00cbde86`
2. **权限控制**：数据库集合权限设置为「仅创建者可读写」，确保用户数据安全
3. **OpenID 获取**：云函数中通过 `cloud.getWXContext().OPENID` 自动获取用户标识
4. **日期格式**：统一使用 `YYYY-MM-DD` 格式存储日期
5. **错误处理**：所有云函数调用需添加 try-catch 错误处理
